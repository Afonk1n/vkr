# Статус пояснительной записки и сравнение с референсом

Документ ведется как рабочая карта перед ручной проверкой. Основной актуальный файл: `Афонькин М.А. ВКР черновик v5.docx`.

## 1. Текущий результат

- Актуальный DOCX пересобран и отрендерен через artifact-tool.
- Рендер: 49 страниц.
- Содержательных таблиц: 12.
- Вставленных схем: 5.
- Заглушки под будущие реальные скриншоты сохранены там, где интерфейс еще может меняться.
- Служебные страницы до содержания временно убраны, чтобы позже подставить правильную университетскую "рыбу".

## 2. Что уже усилено

- Добавлены таблицы функциональных требований, маршрутов frontend, REST API и основных сущностей базы данных.
- Добавлены сценарные таблицы: просмотр ленты, создание рецензии, редактирование профиля, модерация, добавление альбома и треков.
- Добавлена таблица результатов проверки основных сценариев.
- Расширены выводы по главам и заключение.
- Добавлен блок про DevOps-инструменты: Docker Compose, GitHub Actions, production-like smoke test.
- Таблицы приведены к более спокойному оформлению: подпись сверху, без цветных заливок, повторяемая строка заголовка, левое выравнивание текста внутри ячеек.

## 3. Что показал референс Кустубаева

- В работе много сценарных таблиц формата "Действия пользователя / Реакция системы".
- Есть отдельные таблицы по маршрутам клиентского приложения и серверного API.
- Активно используются скриншоты интерфейса, диаграммы и приложения с листингами.
- Третья глава опирается на конкретные результаты проверки, а не только на описание реализации.
- Объем добирается не водой, а таблицами, рисунками, сценариями и приложениями.

## 4. Ближайшие улучшения до финальной версии

1. Добавить реальные скриншоты после стабилизации интерфейса: лента, профиль, альбом, трек, админка, глоссарий, форма добавления альбома.
2. Вставить скриншот успешного GitHub Actions workflow и, возможно, состояние Docker Compose.
3. Добавить приложения с листингами: REST API, модели данных, seed/demo-data, compose, workflow.
4. Проверить оформление переносов больших таблиц по СМК. Сейчас строка заголовка таблицы настроена на повтор, но явная надпись "Продолжение таблицы" пока не добавлена.
5. После ручной проверки вернуть титульные/служебные страницы по официальной форме.
6. Добрать объем до 60+ страниц за счет реальных скриншотов, приложений и точечных технических фрагментов.

## 5. Замечания для ручной проверки

- Формулировки уже очищены от черновых фраз вида "целесообразно разместить" и "может быть использовано в работе".
- Средняя оценка пользователя в тексте не используется как фактор уровня, потому что это логически неверно: уровень должен зависеть от активности и реакции сообщества.
- Таблицы сейчас читаются лучше, но после финальных требований СМК можно отдельно решить, оставлять ли кегль 12 внутри таблиц или приводить все к 14.
- Содержание пока черновое и не обновлено автоматически.
