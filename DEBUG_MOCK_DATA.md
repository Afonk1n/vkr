# Отладка моковых данных

## Проблема
Данные не отображаются на главной странице после deployment на Vercel.

## Что проверить

### 1. Консоль браузера
Откройте консоль разработчика (F12) и проверьте:
- Есть ли ошибки при импорте `mockData.js`
- Что выводится в `console.log` с пометкой "Mock data"
- Есть ли ошибки JavaScript

### 2. Проверка файла mockData.js
Убедитесь, что файл `frontend/src/data/mockData.js`:
- Существует в репозитории
- Попадает в сборку (не игнорируется .gitignore)
- Правильно экспортирует данные

### 3. Проверка путей к изображениям
В моковых данных используются пути:
- `/preview/basta1.jpg`
- `/preview/basta2.jpg`
- и т.д.

Убедитесь, что эти файлы находятся в `frontend/public/preview/` и попадают в сборку.

### 4. Проверка состояния компонента
В консоли должны быть логи:
- "Mock data check:" - проверка импорта
- "Loading mock data..." - начало загрузки
- "Setting mock data:" - установка данных
- "Mock data loaded successfully" - завершение загрузки
- "HomePage state:" - текущее состояние компонента

## Возможные решения

### Если данные не импортируются:
1. Проверьте, что файл `mockData.js` существует
2. Убедитесь, что экспорты правильные (`export const`)
3. Проверьте, что файл не игнорируется при сборке

### Если данные загружаются, но не отображаются:
1. Проверьте условия рендеринга в `HomePage.js`
2. Убедитесь, что `loading` становится `false`
3. Проверьте, что массивы не пустые

### Если изображения не загружаются:
1. Проверьте пути к изображениям в `public/preview/`
2. Убедитесь, что файлы попадают в сборку
3. Проверьте консоль на ошибки 404 для изображений

## Временное решение для отчёта

Если данные всё ещё не отображаются, можно:
1. Добавить данные прямо в `HomePage.js` (временно)
2. Использовать статические данные без импорта
3. Проверить, что сборка на Vercel включает все файлы

## Команды для проверки локально

```bash
cd frontend
npm run build
# Проверьте, что в build/static/js/ есть файлы с mockData
```
